自定义数据属性

HTML5允许给元素指定非标准的属性，但要使用前缀data-以便告诉浏览器，这些属性既不包含与渲染有关的信息，也不包含元素的语义信息。 除了前缀，自定义属性对命名是没有限制的，data-后面跟什么都可以。下面是一个例子:

自定义属性

<div id="myDiv" data-appId="12345" data-myname="Matt">自定义属性</div>

定义了自定义数据属性后，可以通过元素的dataset属性来访问。dataset 属性是一个DOMStringMap的实例，包含一组键/值对映射。 元素的每个 data-name属性在 dataset 中都可以通过 data-后面的字符串作为键来访问 (例如，属性 data-myname、data-myName 可以通过 myname 访问, 但要注意 data-my-name , data-my-Name 要通过myname 来访问) “连字符（-）是‘大写开关’，第一个 - 前全小写，每个 - 后面的第一个字母大写” (简单来是就是驼峰命名) 下面是一个 使用自定义数据属性的例子 

      // 本例中使用的方法仅用于示范

    let div = document.getElementById('myDiv')

    // 取得自定义数据属性的值
    let dataName = div.dataset
    console.log(dataName)
    console.log(dataName.myname)
    console.log(dataName.appid)
    // DOMStringMap {appid: '12345', myname: 'Matt'}

    // 有 'myname' 吗 ?
    if (div.dataset.myname) {
      console.log(`Hello , ${div.dataset.myname}`)
    }

    // Hello Arrebol

自定义数据属性非常适合需要给元素附加某些数据的场景，比如链接追踪和在聚合应用程序中标识页面的不同部分。另外 , 单页应用程序框架也非常多地使用了自定义数据属性
