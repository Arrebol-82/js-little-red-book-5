Node类型

DOMLevel1描述了名为，这个接口是所有DOM节点都必须实现的 。Node接口在javascript 中被实现为Node类型 , 所以浏览器中都可以直接访问这个类型。在 javascript 中 , 所有节点类型都继承Node类型 , 因此 所有类型都共享相同的基本属性和方法

每个节点都有 noderype 属性，表示该节点的类型。节点类型由定义在Node 类型上的12个数值常量表示:

  • Node.ELEMENT_NODE(1) 元素节点 (div , p)
  • Node.ATTRIBUTE_NODE(2) 属性节点
  • Node.TEXT_NODE(3) 文本节点 (标签里面的文字)
  • Node.CDATA_SECTION_NODE(4) XML 特有的节点类型
  • Node.ENTITY_REFERENCE_NODE(5) 废弃啦主要是用于XML实体
  • Node.ENTITY_NODE(6) 跟 5 是一样的
  • Node.PROCESSING_INSTRUCTION_NODE(7) 处理指令节点
  • Node.COMMENT_NODE(8) 注释节点 常用
  • Node.DOCUMENT_NODE(9) 文档节点 核心
  • Node.DOCUMENT_TYPE_NODE(10) 文档类型节点
  • Node.DOCUMENT_FRAGMENT_NODE(11) 文档片段节点 高频使用
  • Node.NOTATION_NODE(12) “非 XML 数据的格式”（如图片、音频的格式说明）
  • 每个节点都是跟后面的数字是一一对应的

节点类型可通过与这些常量比较来确定 , 比如:

    const SomeNode = document.getElementById('someNode')
    console.log(SomeNode.nodeType); 1 跟上面的表是一一对应的

    if (SomeNode.nodeType == Node.ELEMENT_NODE) {
      alert('Node is an element')
    }

这个例子比较了someNode.nodeType与Node.ELEMENT_NODE 常量。如果两者相等，则意味着 someNode 是一个元素节点。

浏览器并不支持所有节点类型。开发者最常用到的是元素节点和文本节点。本章后面会讨论每种节点受支持的程度及其用法。( 这可太棒啦 , 原来是会教我的 )
