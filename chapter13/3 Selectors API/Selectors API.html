Selectors API

Javascript库中最流行的一种能力就是根据CSS选择符的模式匹配DOM元素 。比如 , JQuery 就完全以css选择符查询DOM 获取元素引用 , 而不是使用getELementById() 和 getElementsByTagName()

Selectors API (参见W3C网站上的SelectorsAPILevel 1 ) 是W3C推荐标准，规定了浏览器原生支轴CSS查询API。支持这一特性的所有JavaScript库都会实现一个基本的CSS解析器,然后使用已有DOM方法搜索文档并匹配目标节点。 虽然库开发者在不断改进其性能，但JavaScript代码能做到的境有限。通过浏览器原生支持这个API，解析和遍历DOM树可以通过底层编译语言实现，性能也有7数量级的提升。

SelectorsAPILevel1的核心是两个方法:querySelector()和querySelectorA11()。在兼容浏器中，Document 类型和Element 类型的实例上都会暴露这两个方法。

Selectors API Level 2规范在Element 类型上新增了更多方法，比如 matches()、find()和 findA11()。不过，目前还没有浏览器实现或宣称实现find()和findA11()。

追求性能 / 动态同步用 getElement 系列，追求灵活 / 效率用 querySelector 系列—— 后者虽然性能稍差，但在绝大多数业务场景下，这点性能差异完全可以忽略，开发效率的提升更重要。
