类字段的声明

鉴于在构造函数中为每个实例的初始化成员赋值是非常常见的事 , ECMAscript增加类字段声明的快捷方式

      class PersonwithConstrctor {
        constructor() {
          this.friendCount = 6
        }
      }
  
      let a = new PersonwithConstrctor()
      console.log(a.friendCount);
  
      class PersonwithClassFields {
        friendCount = 8
      }
  
      let b = new PersonwithClassFields()
      console.log(b.friendCount);
  
      console.log('如果定义了构造函数,则在构造函数中可以访问类字段声明');
   
      class Person {
        friendCount = 0
        constructor() {
          console.log(this.friendCount);
        }
      }
  
      // console.log(new Person);
      new Person  // 0
  
      console.log('也可以只声明而不初始化');
  
      class Per {
        friendCount
        constructor() {
          console.log(this.friendCount);
        }
      }
  
      // console.log(new Per);
      new Per // undefined
类字段其实除了在没有办法接收参数 , 其他都是和构造函数没什么区别 , 但是一个是在构造函数前初始化 , 一个是在构造函数里面初始化 , 就没啥区别了 , 最后就是大多数情况下用类字段 , 因为方便 , 还有就是 class { age = 18 } 这样也是会初始化的 ,所以直接写在 { 里面的代码会进行初始化 } 还有 { constructor( ) { } } 里面的代码也会 ,

类字段会将里面的值自动绑定上this,而且也不需要再写this , 在写的话就会报错 , 即为了统一 ,还为了方便
